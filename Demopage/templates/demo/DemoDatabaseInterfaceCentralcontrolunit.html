Database

De Database bevat de gegevens van de apparaten en de gegevens die door de Central Control, Optimalisatie, Interface en Communicatie zijn verstuurd. Deze gegevens kunnen daarna uitgestuurd worden aan deze 4 componenten. De database bestaat verschillende tabellen die elk gekoppeld zijn aan een klasse. Zo zijn er een klassen huizen, kamers, slimme apparaten, frigo's ... Elke klasse heeft zijn eigen eigenschappen. Een apparaat heeft als eigenschap het verbruik, een naam, de verwachte gebruiksduur enzovoort. Deze klassen worden aan elkaar gelinkt zodat de database een logische structuur krijgt. Zo kunnen de gegevens makkelijk opgevraagd worden. De database is geschreven in Python en gebruikt een yaml bestand om gegevens in te laden en slaat deze op in een sqlite bestand.

Interface

De interface geeft feedback aan de gebruiker en laat deze zijn slimme-energiewoning bedienen. In dit project is er gekozen om de gebruiker te informeren via een website. Volgens de opgave werkt de server met Django en Bootstrap templates zorgen voor een aangename lay-out. Op deze interface ziet een gebruiker het energieverbruik in functie van de tijd. De pagina Vergelijking geeft het verbruik van een normale en een slimme-energiewoning weer. U kan ook zelf de apparaten bedienen en deadline's invoeren waardoor deze taken automatisch uitgevoerd worden aan de goedkoopste electriciteitsprijs. De interface biedt ook de mogelijkheid nauwkeurig het verbruik van een apparaat op te volgen, zodat u grote verbruikers makkelijk identificeert.

Central Control Unit

De Central Control Unit, is de centrale eenheid die de vraagzijde sturing regelt. De werking van de Central Control Unit kan opgedeeld worden in drie stappen: input, verwerking en output. De eerste stap is input, de Central Control Unit verzamelt gegevens van de verschillende bronnen. De stroomprijs, die het aanbod aan energie weerspiegelt. De weersvoorspellingen helpen het aanbod aan hernieuwbare energie, zowel op de markt (windmolenparken) als privé (eigen zonnepanelen) voorspellen. De gebruiker geeft aan tegen wanneer apparaten hun taak moeten uitvoeren. De huizen geven de toestand van hun apparaten weer, zoals bijvoorbeeld de temperatuur in elke kamer. Vervolgens verwerkt de Central Control Unit al deze gegevens en slaat hij deze op in de database. Een optimalisatie algoritme bepaalt de meest efficiente dagindeling en stuurt deze terug naar de Central Control Unit. In de laatste stap stuurt de centrale eenheid de apparaten aan en geeft deze output aan de gebruiker via een interface .

Interface

De interface zorgt voor de feedback aan de gebruiker. In dit project is er gekozen om de gebruiker te informeren via een website. In praktijk zouden er ook andere opties gekozen kunnen worden zoals bijvoorbeeld een app. Zo is het mogelijk om de prijsveranderingen van het net te zien in Vergelijking. Het ligt voor de hand dat je er ook het verbruik kan aflezen. Je kan ook de slimme apparaten inschakelen of melden tegen wanneer het ten laatste geactiveerd moet zijn. De ontwikkeling van deze website gebeurde met behulp van Django. In de opgave van dit project werd het gebruik van Django en Bootstrap opgelegd. Aan de hand van deze frameworken was het makkelijker om de website te realiseren. Bootstrap zorgt voor een lay-out op een efficiënte manier aan de hand van uitgeschreven templates.

Central Control Unit

Bij de Central Control Unit komen alle gegevens samen. Deze worden opgeslagen in de database. Het onderdeel ’Optimalisatie’ verwerkt deze gegevens tot een optimale planning en stuurt deze terug door. Het team ’Communicatie’ zorgt voor het doorsturen zelf. Bij de Central Control komen verschillende gegevens binnen zoals bv weervoorspellingen. Het verwerkt deze gegevens opdat ze toegankelijk zijn voor de optimalisatie.
